
The JSON API provides an easily parseable interface to GGn. This can be useful for scripts and other tools. Below is the list of information available, the arguments that can be passed to each API, the format of the results, and an extended explanation on the results. This is a modified version of the Gazelle API and may not contain all of the same endpoints as other Gazelle sites. If there is an API missing that you would like to see added, post about it here to let us know!

Using the API bestows upon you a certain level of trust and responsibility. Abusing or using this API for malicious purposes is a bannable offense and will not be taken lightly.

Refrain from making more than five (5) requests every ten (10) seconds.

You can view your API usage history here. It will show the first access time for an endpoint by a given key from a given IP. If you do not recognize an entry, report it to staff ASAP.


Authentication

To use the API, you must have an API key.
To obtain API keys, go to your profile settings and scroll down to the API Keys section near the bottom, fill in a key name, select any permissions the key will need for your task at hand, and hit Save Profile. Your API key will be displayed at the top of the page one time. API keys are not visible anywhere and staff are not able to retrieve them if lost. You will have to delete and recreate the API key if you lose it.


Endpoint Permissions

Each API key can be assigned permissions which determine what endpoints that key has access to. Permissions are not needed for API endpoints that return the same results independent of the API key used, such as the search endpoint. Endpoints like the item equip endpoint and the userlog endpoint require the permission(s) noted within their entry.


Outline

All request URLs are in the form: api.php?request=<ACTION>

API keys are to be sent via an X-API-Key header on the HTTP request.

All the JSON returned is in the form:

{
  "status" : "success",
  "response" : {
    // Response data.
  }
}



If the request is invalid, or a problem occurs, the status will be failure. In this case the value of response is undefined. The HTTP response code will also reflect any errors occurred with your request.


Index

URL: api.php

Arguments: None

Permissions: None

Response format:
When sending no request the API will return the current version of the API. This can be parsed by scripts needing to ensure the availability of the new/updated endpoints. This version number will be incremented as future changes are made. We are using semvar versioning and will increment the patch version (last digit) when backwards compatible modifications are made to existing endpoints, the minor version (2nd digit) when new endpoints are added, and the major version (1st digit) when we make breaking API changes.

{
    "status": "success",
    "response": {
        "api_version": "2.1.0"
    }
}

Item Search

URL: api.php?request=items&type=search (alternate type is 'shop')

Arguments:
(All options are optional and you can use multiple in one search)

    search - a query to search (case-insensitive). By default searches just the items name.
    search_more - when true, makes the search additionally query item descriptions and book excerpts.
    category - used to filter by a category. This should be the name of a category (e.g. "New Site Features"). Alternatively this can be set to 'Featured Only' or 'All'.
    item_type - used to filter by item type. This should be a number. The possible types are: 100 - Standard, 2 - Equippable, 3 - Book, 4 - Card, 5 - Pack, 6 - Adventure Club Item.
    cost_type - used to filter by cost type. This should be a number. The possible types are: 100 - Gold, 2 - Upload, 3 - Download, 4 - Donor Points.
    cost_amount - set the max cost of items you want returned.
    in_stock - when true, only returns items that are in stock.
    no_featured - by default the search returns featured results first. When this is set to true it won't prioritize featured items.
    order_by - options include: category, dateadded, name, cost, and itemtype.
    order_way - asc or desc.
    page - page number to display (default: 1)
    limit - the amount of results to show per page (default: 30)



Response format:
The search works the same as on the shop page. All arguments are the same. This only returns item id's. If you want info on those items, use the above item info endpoint.
